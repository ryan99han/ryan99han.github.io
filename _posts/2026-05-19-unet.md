[U-Net: Convolutional Networks for Biomedical Image Segmentation](https://arxiv.org/pdf/1505.04597)

The U-Net is an architecture for convolutional neural networks consisting of an encoder and decoder

![U-Net](/assets/images/unet.png)

## Overview
- Previous architectures of convolutional networks failed to preserve high-resolution details
- In a standard encoder-decoder network, data is compressed into a bottleneck where spatial information is lost in favor of high-level semantic meaning
- The breakthrough with U-Nets is the introduction of the skip connection, which skips the bottleneck by concatenating high-resolution feature maps from the encoding path directly to the feature maps in the decoding path
- The decoder no longer has to guess about high-resolution details, allowing the network to preserve those details

## Encoder
- 64 filters applied on the input image
- Repeated application of:
  - Two $3 \times 3$ convolutions (unpadded), each followed by ReLU
  - $2 \times 2$ max pooling operation with stride 2 for downsampling
  - Double the number of feature maps at each downsampling stack

## Decoder
- Repeated application of:
  - $2 \times 2$ transposed convolution (upsampling)

    ![Transposed Convolution](/assets/images/transposed_convolution.png)

  - Concatenation with cropped feature maps from contracting path
    - Concatenation simply adds more channels or feature maps of the same width and height
    - Preserves high-resolution information of the input image from the encoding path
    - Enables fusion of features so the model can leverage both high-level and low-level information
    - Improves gradient flow by propagating gradients from output layer back to earlier layers
  - Channel reduction via 3D filters of dimensions $W \times D \times C$ (width, depth, channels)
    - Act as weighted combinations of features from multiple channels
  - Two $3 \times 3$ convolutions, each followed by ReLU
